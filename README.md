## Project title: OPTO Finance Risk Radar

## Project Overview: 
Finance Risk Radar is a SaaS-style transaction validation
and anomaly detection system designed for real-estate
finance operations.

When the transaction records are passed into the SaaS product, before loading them into the CSV loader, the system first checks whether all the necessary columns are present in the uploaded dataset. If any required columns are missing, the CSV loader returns the list of missing columns that need to be inserted. Once all required columns are present correctly, the CSV loader creates a DataFrame and loads the transaction data successfully. The CSV loader also provides a summary containing the total number of transaction records present in the dataset along with the number of missing values in each column.

Then we move into the Rule Validation Engine, which is loaded with 40+ validation rules. The main goal of this engine is to detect hard financial and transactional errors. The output of this engine clearly flags the transaction records that violate the business rules, and all flagged records are stored in a structured JSON violation report file.

Then we move to the Anomaly Detection Engine, where transaction records that satisfy all business rules but still appear suspicious are identified and flagged. To detect such anomalies, the system uses statistical techniques such as Z-score and Interquartile Range (IQR), along with machine learning algorithms such as Isolation Forest. All suspicious transaction records detected by this engine are stored in an anomaly report JSON file, and the engine also provides an overview summary of the transactions that have been flagged.

Then we move to the Risk Engine, which generates a risk score and assigns a risk label for each transaction record. This helps the finance teams understand the severity of the flagged transactions and determine the level of investigation required for each one. The generated risk scores and labels are stored in a structured risk report JSON file.

There is also a Report Generator module that converts the outputs generated by the engines into downloadable JSON, CSV, and text summary reports. From the dashboard, users are provided with options to download all generated reports easily.

We also have a Streamlit Dashboard, which acts as the primary interface for the users. The dashboard provides complete visibility into the transaction records, validation outputs, anomaly reports, and risk summaries. It also provides download options for all reports containing the flagged transaction records.

Overall, this SaaS platform acts as an early-warning system for finance teams and helps organizations detect suspicious, inconsistent, and risky transaction records before they impact downstream financial operations.

## Validation Categories present in the Rule validation engine to flag transaction records

### Identity Validation
- Missing transaction IDs
- Duplicate transaction IDs
- Invalid ID formats

### Financial Validation
- Negative amounts
- Refund > collected amount
- Outstanding mismatch

### Date Validation
- Invalid dates
- payment_date < demand_date
- Future dates

### Cross-record Validation
- Same transaction linked to multiple customers
- Duplicate transactions
